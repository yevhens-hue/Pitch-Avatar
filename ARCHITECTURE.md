# Архитектура Pitch Avatar Wizard

## 1. Обзор Системы
Приложение предназначено для создания интерактивных презентаций посредством ИИ-аватаров. Оно позволяет конвертировать статические презентации в диалоговые решения, где аватар озвучивает текст, отвечает на вопросы (через RAG) и собирает аналитику.

## 2. Технологический Стек
- **Frontend**: Next.js (App Router), React, Vanilla CSS (CSS Modules).
- **UI Компоненты**: Кастомные (Vanilla CSS, без Tailwind согласно требованиям).
- **State Management**: React Context / Zustand, React Query для работы с API.
- **Testing (TDD)**: Jest, React Testing Library.
- **Backend / API**: Next.js Route Handlers.
- **База Данных & Auth**: Supabase (PostgreSQL для реляционных данных, Vector Store для RAG, Storage для файлов и Audio/Video).
- **ИИ-интеграции**:
  - **LLM**: OpenAI GPT-4o / Сlaude (генерация ответов).
  - **Avatar & Video**: HeyGen API или D-ID (генерация видео-аватаров).
  - **TTS**: ElevenLabs API или Azure Speech (озвучка слайдов).

## 3. Основные Модули

### 3.1. Модуль Аутентификации и Авторизации
- Регистрация, вход (Email/Пароль + OAuth).
- Ролевая модель (Пользователь, Админ).

### 3.2. Wizard (Мастер Создания Презентации)
- **Шаг 1**: Загрузка файла (PDF, PPTX).
- **Шаг 2**: Выбор аватара (фото-бейзд/видео) и голоса (TTS).
- **Шаг 3**: Установка Knowledge Base (обучающая база из документов для ИИ-ассистента).
- **Шаг 4**: Настройка промпта (роли) ИИ (например: "Ты - HR менеджер").

### 3.3. Редактор Проекта (Advanced Editor)
- **Слайды**: Список и управление слайдами.
- **Сценарий**: Редактор текста для каждого слайда (для TTS).
- **Аватар**: Расположение аватара, выбор стиля.

### 3.4. AI Assistant & RAG (Knowledge Base)
- Загрузка дополнительных документов в хранилище, парсинг (PDF -> Text -> Embeddings).
- Поиск релевантной информации (Vector Search) при вопросах пользователя.
- Генерация ответа через LLM на основе контекста презентации.

### 3.5. Плеер Презентации
- Воспроизведение слайдов + синхронное видео аватара (или аудио + статика).
- Встроенный виджет чата для задавания вопросов во время просмотра.

### 3.6. Аналитика и Дистрибуция
- Отслеживание открытий ссылок.
- Сессии, длительность просмотра, клики и логи чата с ИИ.

## 4. Паттерны Разработки (GSD + TDD)
- **TDD Flow**: Red -> Green -> Refactor. Сначала пишутся юнит-тесты на бизнес-логику (например, шаги визарда, отправка сообщений ИИ) и интеграционные тесты для UI через RTL.
- **GSD Flow**: MVP-подход. Минимум оверинжиниринга. Быстрое закрытие задач, фокус на рабочий продукт сперва моковыми данными (Mock AI, Mock BD), затем замена на реальные интеграции.

## 5. Схема Базы Данных (MVP)
- `users` (id, email, role)
- `projects` (id, user_id, title, status, avatar_id, voice_id)
- `slides` (id, project_id, image_url, script_text, ord)
- `knowledge_documents` (id, project_id, file_url, parsed_text)
- `assignments` (id, project_id, target_audience, url)
- `sessions/analytics` (id, assignment_id, watch_time, interactions)
