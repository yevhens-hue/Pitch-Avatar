# Декомпозиция задач: Слоевое наложение и границы прокрутки чат-панели Sara

Этот документ разбивает PRD на независимые задачи (vertical slices), готовые к реализации.

---

## Задача 1: Создание надежного сэндвича наложения слоев в CSS-модуле

* **Тип:** AFK (Автономная задача)
* **Блокируется:** Отсутствует - можно начинать немедленно
* **Покрываемые истории:** US-1, US-2, US-3, US-4, US-5, US-6

### Что сделать
Применить явные уровни наложения с помощью свойств CSS `position` и `z-index` внутри `ChatPanel.module.css`, чтобы поместить прокручиваемую область сообщений `.messages` между верхней шапкой, центральной секцией аватара и нижней областью ввода.

### Критерии приемки
- [ ] Класс `.header` имеет относительное позиционирование (`position: relative`) и `z-index: 3`.
- [ ] Класс `.avatarSection` имеет относительное позиционирование (`position: relative`) и `z-index: 2` для нахождения поверх прокручиваемых строк сообщений с активными анимациями `transform`.
- [ ] Класс `.inputArea` имеет относительное позиционирование (`position: relative`) и `z-index: 3`.
- [ ] Прокручиваемые сообщения полностью и аккуратно исчезают под сплошным фоном `.avatarSection` вместо отображения поверх него.
- [ ] Все концентрические кольца, анимации пульсации и интерактивные кнопки внутри секции аватара работают штатно и сохраняют свой первоначальный дизайн.

## Задача 2: Проведение визуальной проверки и запуск регрессионного тестирования

* **Тип:** AFK (Автономная задача)
* **Блокируется:** Задача 1

### Что сделать
Проверить визуальное отображение виджета при различных размерах экрана, чтобы гарантировать полное отсутствие наложения текста на кольца аватара или границы шапки/области ввода. Запустить автоматизированные тесты для подтверждения стабильности всех моков и хранилищ данных.

### Критерии приемки
- [ ] Визуально подтверждено, что прокручиваемый текст идеально обрезается на границе `.avatarSection`.
- [ ] Выполнен запуск тестов `npx jest src/widgets/Sara` (отсутствие регрессий в состояниях виджета).
- [ ] Выполнен полный запуск тестов проекта `npm run test` (100% прохождение всех тестов).
