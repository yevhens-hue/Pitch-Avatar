# 1. Overview
**Epic Name:** Project Templates System  
**Product Owner:** [Product Owner Name]  
**Based on:** [PRD v.1.0] Project Templates  
**Status:** To Do  

## 1.1. Description
Разработка и внедрение системы шаблонов (Templates) для упрощения и ускорения процесса создания презентаций пользователями. Эпик охватывает отображение шаблонов на главной странице (Home Page) в виде отдельного блока, создание выделенного раздела со всеми шаблонами, интеграцию выбора шаблона в редактор презентаций, а также обновление существующей панели администратора для управления базой шаблонов и логикой их отображения.

## 1.2. Business Value
- **Ускорение онбординга:** Снижение времени на создание первой презентации за счет использования готовых шаблонов.
- **Повышение конверсии:** Увеличение количества успешно созданных презентаций (уменьшение drop-off rate на этапе старта с чистого листа).
- **Гибкость управления:** Администраторы получают удобный инструмент для продвижения определенных шаблонов на главной странице без необходимости привлечения разработчиков.

## 1.3. Product Structure & Flow
Пользователь может начать создание презентации из трех точек:
- Блок **"Templates"** на Home Page (кураторская подборка от админа).
- Отдельная страница **"Templates"** (полный каталог).
- Внутри самого редактора при старте нового проекта.

Администратор управляет расширенными свойствами шаблонов через уже существующий раздел админ-панели **"Шаблоны презентаций"**.

# 3. User Stories (US)

## US-01: Управление шаблонами (Admin CRUD - Project Templates)
**Как** администратор, **я хочу** управлять шаблонами в едином разделе `Project Templates` (без отдельного раздела `Source Projects`), **чтобы** интерфейс был проще и понятнее.

**Acceptance Criteria:**
- В админке остается только один раздел **"Project Templates"** (сущность и вкладка `Source Projects` убирается/объединяется).
- В таблице шаблонов добавляется колонка **Type** (Тип шаблона).
- **КРИТИЧЕСКИ ВАЖНО (Права и защита данных):** При создании шаблона на базе проекта пользователя, система должна делать **физическую копию** этого проекта для администратора. 
  - Это решает проблему прав доступа: админ становится владельцем копии и получает к ней полный доступ.
  - Это решает проблему потери данных: если пользователь удалит свой оригинальный проект, шаблон не сломается и не исчезнет.
- Механику получения доступа админа к пользовательскому проекту для снятия этой копии необходимо проработать с бэкендом (Юра Чернобай), либо реализовать ручную выдачу прав на этапе MVP.

## US-02: Настройка отображения шаблонов на Home Page (Admin)
**Как** администратор, **я хочу** управлять списком и порядком вывода шаблонов на главной странице пользователей.

**Acceptance Criteria:**
- Шаблон выбирается из списка и сразу выставляется на Home Page на нужную позицию с учетом тегов.
- Отказ от жестко закрепленных мест ("прибивания гвоздями"). В будущем (или при масштабировании) будет добавлен динамический фильтр "наиболее часто используемые" (Most popular). 
- На данный момент (пока шаблонов ~8) достаточно текущих тегов и поиска, сложная фильтрация не критична.

## US-03: Блок шаблонов на Home Page (User)
**Как** пользователь, **я хочу** видеть подборку рекомендованных шаблонов на главной странице, **чтобы** быстро начать создание презентации.

**Acceptance Criteria:**
- На Home Page выводится отдельный блок **"Templates"**.
- В блоке отображаются только те шаблоны, которые отмечены администратором для показа на главной.
- Порядок вывода строго соответствует настройкам из админ-панели.
- При клике на шаблон инициируется создание презентации.

## US-04: Страница всех шаблонов (User)
**Как** пользователь, **я хочу** иметь доступ к полному списку всех доступных шаблонов в отдельном разделе, **чтобы** выбрать наиболее подходящий вариант.

**Acceptance Criteria:**
- Доступна отдельная страница (например, `/templates`).
- Шаблоны выводятся в виде сетки карточек (превью, название, описание).
- Присутствует пагинация или бесконечный скролл.
- Есть функционал поиска по названию шаблона.

## US-05: Выбор шаблона в редакторе (User)
**Как** пользователь, **я хочу** иметь возможность выбрать шаблон непосредственно в интерфейсе редактора при создании новой презентации, **чтобы** не возвращаться на главную страницу.

**Acceptance Criteria:**
- При старте нового проекта в редакторе предлагается выбор: "Создать пустую" или "Выбрать шаблон".
- При выборе шаблона новая презентация инициализируется данными, структурой и слайдами из эталонного проекта, привязанного к шаблону.

# 4. Implementation Phases

### Phase 1: Database & Admin Updates (Спринт 1)
**Фокус:** Обновление схемы данных и расширение существующего раздела в админке.
- [ ] Расширение схемы БД для сущности `Template` (добавление полей Description, Selected Project ID).
- [ ] Обновление API для CRUD операций с шаблонами.
- [ ] Доработка интерфейса в существующем разделе **"Шаблоны презентаций"** в админ-панели (добавление новых полей в форму создания/редактирования).
- [ ] Реализация привязки "эталонного проекта" к шаблону и настройка прав доступа (чтобы только админ мог редактировать оригинал).

### Phase 2: Home Page & Admin Display Controls (Спринт 2)
**Фокус:** Вывод на главной странице и настройка порядка.
- [ ] Добавление полей `is_on_homepage` и `order` в БД (или аналогичной логики).
- [ ] Реализация UI в админке для управления выводом и сортировкой на Home Page (столбцы в таблице или поля в форме).
- [ ] Создание API для получения отсортированного списка шаблонов для Home Page.
- [ ] Верстка и интеграция блока **"Templates"** на Home Page пользователя.

### Phase 3: Templates Page & Editor Integration (Спринт 3)
**Фокус:** Каталог шаблонов и интеграция с редактором.
- [ ] Создание страницы `/templates` с пагинацией и поиском.
- [ ] Доработка функционала клонирования проектов: создание пользовательской копии на основе ID "эталонного проекта" шаблона.
- [ ] Внедрение UI выбора шаблона в процесс создания презентации внутри редактора.

### Phase 4: QA, Polish & Release (Спринт 4)
**Фокус:** Тестирование и запуск.
- [ ] Интеграционное тестирование флоу создания из шаблона из всех трех точек (Home Page, `/templates`, Editor).
- [ ] Проверка прав доступа: пользователи не должны иметь возможности как-либо сломать "эталонные проекты".
- [ ] Настройка событий аналитики в PostHog (создание презентации из шаблона, просмотр списка).
- [ ] Включение функционала для пользователей.
