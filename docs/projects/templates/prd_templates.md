## 🎯 Цели
*   **Повышение уровня активации:** Сократить время до первой публикации (Time to First Publication, TTFP) на 50% за счет устранения необходимости создавать презентацию с нуля.
*   **Увеличение использования платформы:** Достичь того, чтобы 40% новых презентаций создавались на основе встроенных шаблонов в первом квартале после запуска.
*   **Переход на нативные объекты:** Перевести систему с плоских PPTX файлов на динамические JSON-структуры, чтобы включить мгновенный перевод и редактирование текста.
*   **Увеличение использования перевода:** Рост метрики перевода контента, так как нативные объекты позволяют переводить весь слайд в один клик (в отличие от статичных PPTX).
*   **Монетизация (Upsells):** Использование специализированных экспертных шаблонов (например, EU AI Act, Compliance) как премиального контента, доступного на высоких тарифах или за разовую плату.

## 📖 Бизнес-контекст
> [!IMPORTANT]
> На данный момент новые пользователи Pitch Avatar часто сталкиваются с пугающим «чистым холстом» или необходимостью загружать полностью готовый PPTX. Это вызывает трение, откладывает активацию и лишает платформу эффекта мгновенного «Wow!».
> 
> **Ключевой запрос от бизнеса (Chief Expert View):**
> Шаблоны не должны быть просто красивой картинкой. Это должен быть **механизм активации**, как у Gamma или Synthesia. Пользователь должен в один клик получить готовую бизнес-структуру со сценарием для ИИ-аватара и редактируемым текстом. Для этого мы уходим от формата файлов `.pptx` к **нативным JSON-объектам** (со слоями для фона, текста и изображений).
> 
> **AI Team Review Note:** Мы должны фокусироваться не на абстрактном креативе, а на **экспертном B2B контенте** и **мгновенной мультиязычности** (голос + текст), что является нашим главным преимуществом перед Gamma.

## 🛠 Требования

### Экран 1: Галерея шаблонов
| Поле | Требование |
| :--- | :--- |
| **Описание** | Интерфейс на панели управления (`/main/presentation-templates`) для выбора одного из 12 базовых бизнес-сценариев. |
| **Текущий статус** | • Базовая таблица в админ-панели и визуальная галерея карточек уже существуют по пути "Start from Template".<br>• Реализована сущность **Product Types** (Типы продуктов), которая выступает в роли категорий. |
| **Критерии приемки** | • Синхронизировать бизнес-терминологию: переименовать "Product Types" в "Categories" для 12 целевых бизнес-категорий.<br>• Внедрить кнопку **«Translate & Voice»** прямо в карточку шаблона для мгновенной локализации. |
| **Аналитика** | `template_gallery_viewed`, `template_selected`, `category_filtered`. |

### Экран 2: Редактирование нативного шаблона
| Поле | Требование |
| :--- | :--- |
| **Описание** | Взаимодействие с выбранным шаблоном в редакторе. |
| **Принцип Constrained UI** | Пользователь может редактировать текст и менять изображения, но **макет (layout), положение фона и аватара заблокированы**. Это гарантирует сохранение профессионального вида презентации. |
| **Критерии приемки** | • **Фон (Background):** Фоновое изображение жестко отделено от контента и заблокировано.<br>• **Z-Index:** Текст и изображения наслаиваются корректно (абсолютное позиционирование).<br>• **Локализация:** Встроенный инструмент перевода текстовых блоков вместе со сценарием аватара (`avatarScript`). |
| **Аналитика** | `template_text_edited`, `template_translated`, `background_changed`. |

### Модуль 3: Улучшенный импортер PPTX (Админка)
| Поле | Требование |
| :--- | :--- |
| **Описание** | Инструмент для преобразования legacy PPTX-шаблонов в нативные объекты с помощью LLM. |
| **Критерии приемки** | • **Hybrid Extraction:** Использование XML для текста и Vision LLM (Claude 3.5 Sonnet) для понимания визуальной композиции слайда.<br>• **Flattening Fallback:** Сложные элементы (диаграммы, SmartArt) рендерятся как статичные PNG, в то время как текст остается нативным.<br>• **Z-index Heuristic:** Эвристический алгоритм определения порядка слоев на основе XML-структуры. |
| **Аналитика** | N/A (Внутренний тул) |

## 📂 Контентная стратегия (12 столпов)
Мы запустим 12 экспертных шаблонов:
1. **Онбординг:** Интеграция сотрудников.
2. **Внутренние коммуникации:** Апдейты компании.
3. **Демо продукта:** Продажи.
4. **Корпоративное обучение:** Образование.
5. **HR-ассистент:** Ответы на вопросы.
6. **CustDev Interviews:** Исследования.
7. **Виртуальный рекрутер:** Скрининг.
8. **Служба поддержки:** Trouble-shooting.
9. **Виртуальный менеджер по партнерам:** Обучение.
10. **EU AI Act Training:** Регуляторный контент.
11. **Борьба с коррупцией:** Комплаенс.
12. **Investor Update:** Регулярные отчеты для инвесторов (добавлено по рекомендации маркетинга).

## 🚫 Исключено из этой версии
*   **Сложные анимации PPTX:** Фаза 1 фокусируется на статичных нативных объектах.
*   **Свободное перетаскивание:** В рамках **Constrained UI** свободное перемещение базовых элементов заблокировано для сохранения стиля.
