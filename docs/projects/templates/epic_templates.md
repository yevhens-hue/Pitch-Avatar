# Epic: Интеграция нативных шаблонов и импортер
**Роль:** AI Инженер
**Проект:** Pitch Avatar
**Дата:** Май 2026

## Обзор
Этот Epic определяет инженерные усилия, необходимые для перевода шаблонов Pitch Avatar из плоских файлов PPTX в нативную объектную структуру JSON. Он специально направлен на решение трех технических запросов CEO: разделение фона, импорт PPTX на базе LLM с ограничениями и управление порядком объектов (z-index).

## Критерии готовности (DoD)
*   **Функционал:** Пользователь может выбрать один из 11 базовых шаблонов, и редактор загружает его как нативные текстовые и графические объекты.
*   **Импортер:** Администратор может загрузить PPTX, и бэкенд-конвейер LLM преобразует его в схему слайда JSON (фон + объекты).
*   **UI:** Объекты наслаиваются правильно (z-index) и могут быть индивидуально выбраны и отредактированы в интерфейсе (даже если редактор базовый).
*   **Локализация:** Нативные текстовые объекты внутри шаблона могут быть переведены с помощью существующего API перевода.

## Пользовательские истории и технические задачи

### 1. Архитектура фона слайда (Задача 1 от CEO)
*Как пользователь, мне нужно, чтобы фон шаблона был отделен от текста, чтобы изменение текста не нарушало дизайн.*

*   **Задача 1.1: Обновление схемы базы данных**
    *   Обновить модель `Slide` в базе данных, включив объект `background`: `{ type: 'image' | 'color', value: string }`.
*   **Задача 1.2: Обновление рендеринга фронтенда**
    *   Изменить компонент рендеринга слайдов (например, `SlideViewer.tsx`), чтобы объект `background` рендерился как CSS `background-image` или `background-color` в контейнере с самым низким z-index.
*   **Задача 1.3: Загрузчик фона для админа**
    *   Позволить администраторам, создающим шаблоны, загружать изображение, специально обозначенное как «Фоновое изображение».

### 2. Ограниченный импортер PPTX на базе LLM (Задача 2 от CEO)
*Как администратор, я хочу автоматически преобразовывать существующие шаблоны PPTX в нативные объекты Pitch Avatar с допустимыми упрощениями дизайна.*

*   **Задача 2.1: Микросервис парсинга PPTX**
    *   Реализовать бэкенд-скрипт (Python/Node) для извлечения XML-данных и изображений слайдов из файла `.pptx`.
*   **Задача 2.2: Конвейер интеграции LLM**
    *   Интегрировать API Claude 3.5 Sonnet или GPT-4o Vision.
    *   Отправлять изображение слайда + извлеченный текст XML с промптом для возврата JSON-массива `TextObjects`.
*   **Задача 2.3: Логика отката (Fallback) импортера**
    *   Реализовать логику: если объект в XML помечен как `SmartArt`, `Chart` или сложная сгруппированная фигура, отрендерить эту область в `.png` и импортировать ее как стандартный `ImageObject`, а не пытаться парсить нативно.
*   **Задача 2.4: Маппинг данных**
    *   Сопоставить JSON-вывод LLM со схемой `Slide Schema` Pitch Avatar и сохранить в базу данных.

### 3. Z-Index и порядок объектов (Задача 3 от CEO)
*Как пользователь, мне нужно, чтобы изображения и текст перекрывались правильно в соответствии с оригинальным дизайном шаблона.*

*   **Задача 3.1: Свойство Z-Index**
    *   Убедиться, что массив `objects` в `Slide Schema` включает целое число `zIndex`.
*   **Задача 3.2: Абсолютное позиционирование на фронтенде**
    *   Обновить холст редактора фронтенда. Сопоставить `objects` с абсолютно позиционированными элементами HTML.
    *   Применить `style={{ zIndex: object.zIndex }}`, чтобы обеспечить правильное перекрытие.
*   **Задача 3.3: Логика Z-Index в импортере**
    *   Обновить импортер PPTX (Задача 2.1) для присвоения инкрементного `zIndex` (1, 2, 3...) на основе порядка рендеринга элементов XML (снизу вверх).

### 4. Галерея шаблонов и наполнение
*Как пользователь, я хочу видеть 11 высококачественных шаблонов на своей панели управления.*

*   **Задача 4.1: Наполнение базы данных**
    *   Создать 11 категорий шаблонов в базе данных.
*   **Задача 4.2: Интеграция UI галереи**
    *   Подключить интерфейс `/main/presentation-templates` для получения данных из новой таблицы нативных объектов вместо устаревшего списка PPTX.

## Зависимости и риски
*   **Риск:** Галлюцинации LLM при извлечении координат из PPTX.
*   **Смягчение:** Мы будем предоставлять LLM точные данные об ограничивающих рамках из XML, используя Vision-модель только в качестве вторичного подтверждения, а не полагаясь чисто на преобразование изображения в координаты.
*   **Зависимость:** Текущий UI редактора должен поддерживать абсолютное позиционирование нативных объектов. Если это не так, потребуется минимальный рефакторинг области холста.
