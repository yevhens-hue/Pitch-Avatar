## 🔄 Interaction Logic: Stonly Checklist ↔ Stonly Guides

**Концепция:**
*   **Welcome гайд (Stonly)** — обзор "вот что умеет продукт для твоей цели" → CTA переводит на экран. Это "посмотри и пойми" (3 шага).
*   **Чеклист + туры (Stonly)** — конкретные действия которые нужно выполнить. Пункты чеклиста не дублируют гайд, они его продолжают. Пункты чеклиста кликабельны и запускают интерактивные туры (Tooltips).

**High-Level Flow:**
1. Welcome гайд Step 3 → CTA кнопка.
2. Открывается нужный экран продукта.
3. Чеклист появляется (bottom right). Пункт 1 уже ☑ (т.к. CTA привел к первому действию).
4. Пункты 2-5 ○ — пользователь кликает → Stonly Guide (Tooltip tour) ведет руками по реальному интерфейсу.

### 💻 Техническая спецификация интеграции (Ticket)

**Цель:** пользователь кликает на пункт ○ в Stonly-чеклисте - приложение автоматически запускает нужный Stonly Guide прямо в интерфейсе продукта.

**Флоу:**
1. Пользователь кликает на пункт ○ в чеклисте.
2. JS-обработчик читает `data-tour-id` с элемента.
3. Если не на нужном экране — навигация туда через `router.push()`.
4. После загрузки экрана (1500ms) — запуск гайда через `StonlyWidget('openGuide', { guideId })`.
5. Пользователь выполняет действие в продукте (Activation event) → пункт становится ☑.

**Что реализовать:**
- JS-обработчик клика на пунктах чеклиста (MutationObserver + EventListeners).
- Навигация на нужный экран если пользователь не там.
- Запуск гайда через `StonlyWidget('openGuide', ...)`.
- Аналитика в PostHog: `stonly_guide_started`, `stonly_guide_completed`, `stonly_guide_step_reached`.

**Критерии приёмки интеграции:**
- [x] Клик на ○ запускает нужный гайд.
- [x] Если не на нужном экране — сначала навигация, потом гайд.
- [x] Клик на ☑ — гайд не запускается.
- [x] После выполнения действия пункт становится ☑.
- [x] Все события аналитики приходят в PostHog с правильными параметрами.
- [ ] 7 гайдов протестированы на соответствующих экранах.

**Зависимости:**
- Маркетинг настроил чеклист с разметкой (или тексты совпадают с маппингом).
- Маркетинг создал интерактивные гайды (Tooltips) в Stonly и передал ID.
- В проекте установлен сниппет Stonly Widget.

## ТЗ Dev Stonly Checklist ↔ Stonly Guides интеграция

**Цель:** пользователь кликает на пункт ○ в Stonly-чеклисте - приложение автоматически запускает нужный Stonly Guide (Tooltip), подсвечивая нужные кнопки и элементы.

### 1. Описание
Чеклист показывает задачи для активации. Каждый пункт ○ при клике должен запускать интерактивный гайд, который ведет пользователя за руку.

1. Пользователь кликает на пункт ○ в чеклисте.
2. JS-обработчик перехватывает клик и читает `data-tour-id`.
3. При необходимости происходит навигация на нужный экран.
4. `StonlyWidget('openGuide', { guideId })` запускает гайд.
5. Гайд подсвечивает UI-элементы через Tooltips.
6. Действие завершено — пункт чеклиста становится ☑.

### 2. Требования

#### 2.1 Разметка пунктов чеклиста
Тексты пунктов в Stonly должны совпадать с ключами в коде (`TOUR_MAP`) или содержать `data-tour-id`.

#### 2.2 JS-обработчик клика
Слушает клики на `.stonly-checklist-item`, читает атрибуты и запускает гайд через Stonly Widget API.

#### 2.3 Маппинг Guide ID → экран
| Tour ID | Экран для навигации | Используется в чеклисте | Stonly Guide ID |
| :--- | :--- | :--- | :--- |
| `create-chat-avatar` | `/chat-avatar/create` | Чеклист 2 (Chat), 5 (Fallback) | `GUIDE_ID_HERE` |
| `setup-scenario` | `/chat-avatar/create` | Чеклист 2 (Chat) | `GUIDE_ID_HERE` |
| `test-avatar` | `/chat-avatar/create` | Чеклист 2 (Chat) | `GUIDE_ID_HERE` |
| `create-video` | `/create` | Чеклист 1 (Avatar), 5 (Fallback) | `GUIDE_ID_HERE` |

### 3. Критерии приёмки
- [x] Клик на пункт ○ запускает нужный Stonly Guide.
- [x] Навигация отрабатывает корректно до запуска гайда.
- [x] Клик на выполненный пункт ☑ не перезапускает гайд.
- [x] События `stonly_guide_started` уходят в PostHog.
