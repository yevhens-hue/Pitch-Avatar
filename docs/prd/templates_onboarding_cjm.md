## 🎯 Goal
Спроектировать систему онбординга для функционала "Шаблоны" (Templates), которая будет использовать стартовый квиз для рекомендации наиболее релевантных шаблонов. 
Система должна бесшовно переводить пользователя от **Welcome-гайда (Stonly)** к **Чеклисту (Stonly)**, который, в свою очередь, будет запускать **Интерактивные туры (Userflow)** для обучения редактированию нативных объектов шаблона.

**Ключевой результат:** Снижение TTFP (Time to First Publication) за счет того, что пользователь сразу видит нужный ему бизнес-шаблон и обучается работе с ним шаг за шагом.

## 🔄 Interaction Logic: Quiz ↔ Stonly ↔ Userflow

**High-Level Flow:**
1. Пользователь заходит в раздел Templates впервые.
2. Появляется **Quiz (Stonly Modal)**: "What are you trying to build today?".
3. Пользователь выбирает вариант. Stonly сохраняет `template_goal` и фильтрует галерею (показывает нужную категорию первой).
4. Запускается **Welcome Guide (Stonly)** (2 шага), показывающий магию нативных объектов.
5. Открывается **Чеклист** (bottom right). Пункт 1 ○: "Выбери шаблон".
6. Пользователь кликает на пункт 1 → **Userflow** подсвечивает карточку шаблона.
7. После выбора открывается редактор. Чеклист переходит к пункту 2 → Userflow подсвечивает текст на холсте.

## 💻 Техническая спецификация интеграции

Интеграция работает по тому же принципу, что и базовый онбординг, но с фокусом на редактор шаблонов.

### 1. Маппинг Tour ID → Экран (Для Userflow)
| Tour ID | Экран для навигации | Сценарий |
| :--- | :--- | :--- |
| `tour_select_template` | `/main/presentation-templates` | Пункт 1: Выбор шаблона (фокус на категории) |
| `tour_edit_native_text` | `/editor/{template_id}` | Пункт 2: Редактирование текста (фокус на canvas) |
| `tour_setup_avatar_script` | `/editor/{template_id}` | Пункт 3: Настройка аватара |
| `tour_translate_template` | `/editor/{template_id}` | Пункт 4: Кнопка перевода (фокус на локализацию) |
| `tour_publish_template` | `/editor/{template_id}` | Пункт 5: Кнопка Share |

### 2. JS-обработчик клика (Переиспользование)
Мы переиспользуем существующий обработчик `StonlyWidget ready` из основного онбординга.
Разметка пунктов чеклиста в Stonly (от маркетинга) должна выглядеть так:
```html
<button class="checklist-item" data-tour-id="tour_edit_native_text" data-screen="current">
  Измени текст
</button>
```

### 3. Аналитические события (PostHog)
Необходимо добавить параметры квиза в воронку:
*   `templates_quiz_viewed`
*   `templates_quiz_answered` (с параметром `template_goal`)
*   `userflow_tour_started` (с параметром `tour_id` из таблицы выше)
*   `template_activation_completed` (когда весь чеклист ☑)

## ✅ Acceptance Criteria

1.  **Quiz Logic:** Квиз корректно отображается при первом входе и сохраняет ответ в свойство пользователя (или local storage для сессии).
2.  **Gallery Filtering:** Галерея шаблонов автоматически подстраивается (сортируется/фильтруется) под ответ из квиза.
3.  **Checklist Integration:** Клик по пунктам чеклиста запускает соответствующие туры `Userflow` без перезагрузки страницы.
4.  **Editor Constraints:** Тур `tour_edit_native_text` корректно подсвечивает именно текстовый слой, учитывая новую логику Z-index и заблокированного фона.
5.  **Analytics:** Воронка от квиза до публикации шаблона прозрачно отслеживается в PostHog.
