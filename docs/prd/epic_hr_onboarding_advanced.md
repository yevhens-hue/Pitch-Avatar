# Эпик: Модуль HR Onboarding & Engagement (Часть 2: Advanced Features)

## Введение
Данный документ описывает вторую фазу (Advanced Features) модуля HR Onboarding, состоящую из 6 дополнительных User Stories (№6 – №11). Эта фаза масштабирует базовый функционал, добавляя групповые операции, глубокую AI-аналитику, умный импорт и контроль сессий.

---

## 1. Новые Сущности: Масштабирование контента и аудитории

### 📚 User Story 6: Courses (Курсы)
- **Суть:** Объединение проектов (Projects) в логические последовательности или наборы.
- **Проблема:** Сейчас для отправки 3 разных презентаций нужно делать 3 разных Assignment. Невозможно посмотреть сквозную аналитику обучения.
- **Реализация:**
  - Новый CRUD интерфейс: пункт меню `Courses` внутри `Projects`.
  - В Assignment (вкладка General) в поле `Content Type` появляется возможность выбрать `Course`.
  - В аналитике появляется возможность смотреть данные агрегированно по курсу или с детализацией (drill-down) по каждому проекту внутри него.

### 👥 User Story 7: Groups (Группы Слушателей)
- **Суть:** Объединение Слушателей (Listeners) в группы.
- **Проблема:** Отправлять контент каждому слушателю индивидуально — долго. Нет сравнения метрик между отделами/потоками.
- **Реализация:**
  - Новый CRUD интерфейс: пункт меню `Groups` внутри `Listeners`.
  - В форме редактирования Listener появляется привязка к Group.
  - В Assignment в поле `Target Type` появляется возможность выбрать `Group`.
  - Аналитика позволяет смотреть агрегированные данные по группе с возможностью разворачивания до конкретного слушателя.

## 2. Глубокая Локализация и Умный Импорт

### 🌐 User Story 8: Assignment Language Settings (Автоперевод)
- **Суть:** Автоматический перевод контента проекта на родной язык Слушателя.
- **Проблема:** Если кандидат из Польши, а проект на английском, ему будет некомфортно. Ручное создание копий проектов под каждый язык убивает время.
- **Реализация:**
  - Во вкладку `General` (при создании Assignment) добавляются два тумблера: `Set Project Language from Listener` и `Add Listener Language and Translate Project`.
  - Если у проекта нет нужного языка, система через существующий механизм перевода автоматически переводит слайды, скрипты, видео и аудио.
  - Внедряется предупреждение о списании кредитов за генерацию перевода.

### 📥 User Story 10: Import/Export Listeners & CV Parsing (LLM)
- **Суть:** Массовое управление базой слушателей и AI-парсинг резюме.
- **Проблема:** Ручной ввод данных кандидатов и сотрудников. Невозможность быстрой выгрузки базы для других систем (до настройки прямой API-интеграции).
- **Реализация:**
  - **Массовые операции:** Импорт/Экспорт таблиц CSV и XLS. Механизм обработки дублей по `email` (Ignore/Override).
  - **AI CV Parsing:** HR может пачкой загрузить PDF/Word файлы резюме. LLM сама достает из них Имя, Email, Должность и Навыки, заполняет профиль Listener, а сам файл прикрепляет в секцию `Documents`.

## 3. Контроль Сессий и Продвинутая AI-Аналитика

### 🎥 User Story 9: Session Control (Video Recording & Summary)
- **Суть:** Запись с веб-камеры слушателя, транскрибация и генерация саммари.
- **Проблема:** При удаленном тестировании кандидат может гуглить ответы. Текстового чата недостаточно для оценки эмоций и "софт-скиллов".
- **Реализация:**
  - Во вкладке `Session` (настройки Assignment) добавляются чекбоксы: `Record Listener Video` и `Generate Session Summary`.
  - **Флоу слушателя:** Перед началом сессии появляется экран запроса разрешений (Consent Screen) на запись камеры.
  - **Результат в Analytics:** Появляются кнопки просмотра видеозаписи, полного текстового транскрипта сессии и выжимки (Summary), сгенерированной LLM на основе чата и аудио.

### 🧠 User Story 11: AI Analysis в Analytics
- **Суть:** LLM-анализ массива результатов и генерация инфографики.
- **Проблема:** Обилие сырых цифр не дает инсайтов (например, "кто из кандидатов склонен к обману" или "какой вопрос заваливает 80% сейлзов").
- **Реализация:**
  - Новая вкладка `AI Analysis` в разделе `Analytics`.
  - Пользователь выбирает нужные Assignments, вводит текстовый запрос (Prompt) и нажимает анализировать.
  - LLM выдает текстовый ответ с инсайтами. Опционально генерируется изображение (инфографика), если выбран соответствующий чекбокс.
