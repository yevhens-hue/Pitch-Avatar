## 🎯 Цели
*   **Автоматизация онбординга и найма:** Устранить ручную отправку ссылок и сбор результатов, заменив их автоматизированными рассылками (Assignments) и цепочками писем с напоминаниями.
*   **Персонализация и трекинг:** Добавить базовую сущность `Listener` для трекинга метрик по конкретному человеку и сбора детальной аналитики.
*   **Интеграция с бизнес-процессами:** Настроить двусторонний обмен данными с ATS/CRM (HubSpot, Salesforce) для бесшовного процесса найма, обучения и продаж.
*   **Новая модель монетизации:** Внедрить биллинг на основе **Listener Seats** (места слушателей) для Enterprise-клиентов по AWS-like (PayPro Global metered) модели.

## 📖 Бизнес-контекст
> [!IMPORTANT]
> Существующий флоу анонимных ссылок (Links) абсолютно не подходит для HR-онбординга и B2B обучения. Невозможно отправлять персонализированный контент, собирать аналитику в разрезе конкретного человека и выставлять счета (Billing). 
> 
> **Ключевой запрос от бизнеса (Chief Expert View):**
> Платформа должна стать нативной частью бизнес-процесса Enterprise-клиентов, а не «еще одной изолированной тулзой». Для этого нужно сфокусироваться на цепочке: `Listener -> Assignment -> Results -> Integrations`. Это расширит применимость продукта на 4 ключевых направления:
> 1. **L&D:** Наставничество и тестирование сотрудников.
> 2. **Sales:** Презентация продуктов и квалификация лидов.
> 3. **Recruitment:** Автоматизированные первичные интервью и скрининг.
> 4. **CustDev:** Масштабная проверка гипотез.

## 🛠 Требования

### 1. База Слушателей и Импорт (Listeners)
| Поле | Требование |
| :--- | :--- |
| **Описание** | Единая CRM-подобная база слушателей, их профилей и прикрепленных документов, с возможностью объединения в группы. |
| **Критерии приемки** | • **Listener CRUD:** Создание профиля (Email, Имя, Должность, Страна, Язык).<br>• **Groups:** Объединение Слушателей в группы для массовых отправок.<br>• **Умный импорт:** AI CV Parsing — HR загружает PDF/Word резюме, LLM достает Имя, Email и Навыки, а сам файл сохраняется в `Documents[]`.<br>• **Массовые операции:** Импорт/Экспорт CSV с обработкой дублей. |
| **Аналитика** | `listener_created`, `cv_parsed`, `group_created`. |
| **User Story** | Как HR, я хочу пачкой загрузить PDF резюме кандидатов, чтобы AI сам извлек их контактные данные, заполнил профили и подготовил их к отправке тестового задания. |

### 2. Рассылки и Автоматизация (Assignments)
| Поле | Требование |
| :--- | :--- |
| **Описание** | Замена ручного шеринга ссылок (Share) на систему автоматизированных задач (Assignments) с контролем дедлайнов. |
| **Критерии приемки** | • **Создание рассылки:** Выбор типа (Listener/Group) и контента (Project/Course).<br>• **Email Schedule:** Настройка до 3-х напоминаний с плейсхолдерами `{{listener_first_name}}`.<br>• **Автолокализация:** Тумблер "Translate to Listener language" для автоматического перевода презентации на родной язык кандидата.<br>• **Тихий режим:** Поддержка "тихих открытий" без лишних уведомлений. |
| **Аналитика** | `assignment_created`, `reminder_sent`, `assignment_completed`. |
| **User Story** | Как рекрутер, я хочу один раз настроить Assignment, чтобы система сама отправила кандидату проект, трижды напомнила ему при игнорировании, и автоматически перевела текст на его язык. |

### 3. Контроль Сессий (Session Control)
| Поле | Требование |
| :--- | :--- |
| **Описание** | Инструменты прокторинга и записи для удаленного тестирования и первичных интервью. |
| **Критерии приемки** | • **Consent Screen:** Запрос разрешений на использование веб-камеры и микрофона перед началом сессии.<br>• **Video Recording:** Запись видео сессии слушателя.<br>• **Транскрибация и Summary:** Автоматическая транскрибация сессии и генерация LLM-саммари по эмоциям и ответам кандидата. |
| **Аналитика** | `session_recorded`, `summary_generated`. |
| **User Story** | Как L&D менеджер, я хочу видеть видеозапись того, как сотрудник проходит тест по Compliance, и читать AI-саммари его ответов, чтобы убедиться в отсутствии списывания. |

### 4. Аналитика и Метрики (Results)
| Поле | Требование |
| :--- | :--- |
| **Описание** | Продвинутая система агрегации метрик, AI-анализа и передачи данных во внешние системы. |
| **Критерии приемки** | • **Агрегация (Result Value):** Вычисление по правилам Last value, Sum, Max, Use LLM (через Prompt).<br>• **AI Analysis:** Отдельная вкладка, где можно попросить LLM проанализировать массив результатов выбранных Assignments и выдать инсайты.<br>• **Интеграции:** Настройка маппинга (`Result X -> App Y -> Entity Z -> Field W`) для передачи данных в CRM/ATS (например, создание Deal/Contact).<br>• **Manual Entry:** Ввод результатов вручную (например, дата успешного найма оффлайн). |
| **Аналитика** | `analytics_viewed`, `ai_analysis_requested`, `integration_sync_success`. |
| **User Story** | Как руководитель продаж, я хочу попросить AI проанализировать результаты тестов всего отдела, чтобы быстро понять, на каком вопросе "валится" 80% моих сейлзов. |

## 🚫 Исключено из этой версии
*   **Матричная структура (Group x Course):** В текущей фазе закладываем только "пустые" таблицы в БД (`groups`, `listener_groups`, `courses`, `course_projects`) для легкой миграции в будущем. Полный функционал LXP (Learning Experience Platform) будет реализован позже.
*   **Глубокое управление группами (Complex Hierarchy):** Сложные вложенные структуры отделов и доступов перенесены на следующие этапы.
*   **Двусторонняя синхронизация с CRM (Inbound Webhooks):** В первом релизе фокус только на отправке данных (Outbound) в ATS/CRM (создание контакта по итогам теста). Прием данных из CRM (создание Listener'а при изменении статуса сделки) перенесен во вторую фазу.
*   **AI Proctoring & OCR:** Распознавание лиц, отслеживание взгляда (anti-cheating) или сканирование резюме-картинок (OCR) не входят в MVP. Запись сессии ведется как обычный видео-файл, а парсинг CV работает только с текстовыми слоями PDF/Word.
*   **Интеграции со Slack/MS Teams:** Уведомления слушателям (Reminders) отправляются строго по Email.
*   **Слияние профилей (Listener Merging):** Механика объединения дубликатов (например, если кандидат сначала подался с личной почты, а затем добавлен с корпоративной) исключена. В v1 это будут разные сущности.
*   **Сверхбольшие документы:** AI CV Parsing будет жестко ограничен по размеру (например, до 5 страниц). Обработка 50-страничных портфолио исключена, чтобы избежать превышения контекстного окна LLM и лишних затрат.
*   **Полный аудит доступности (WCAG 2.1):** Специализированная адаптация интерфейса плеера для слабовидящих кандидатов не гарантируется в первом релизе.

## ❓ Открытые вопросы
1.  **Права доступа Integrations:** Кто имеет право настраивать Integrations? Только Super Admin или Account Admin тоже? *(Рекомендация: Account Admin может настраивать интеграции из предварительно одобренного списка).*
2.  **Оценка трудозатрат Sprint 4:** Насколько реалистична оценка сроков по интеграции Tiered Billing (PayPro Global) вместе с 3rd party API коннекторами? Не стоит ли вынести Webhooks в отдельный спринт?
3.  **Soft/Hard Delete для Listener:** Что происходит с аналитикой при удалении Listener? *(Рекомендация: Soft delete для Listener + статус Archived для Assignment, чтобы не ломать историческую отчетность).*
4.  **GDPR и Retention (Privacy):** Записи сессий (видео) и резюме кандидатов — это чувствительные данные (PII). Должны ли мы внедрять автоматическое удаление (Data Retention Policy) файлов через 30-90 дней для соответствия GDPR?
5.  **Блокировка Seats (Авто-экспирация):** Если слушатель начал Assignment (занял Listener Seat), но забросил его — Seat остается заблокированным навсегда? Нужно ли внедрить автоматический Time-to-Live (TTL) для незавершенных тестов?
6.  **Обработка Overage (Перерасход):** При массовом импорте CSV (например, 150 человек) при лимите в 100 Seats: мы жестко блокируем 50 человек или разрешаем "Soft Overage" с постоплатой через PayPro в конце месяца?
7.  **Триггер биллинга (Billing Trigger):** В какой момент активируется Listener Seat и уходит usage-отчет в PayPro? В момент создания Assignment HR-менеджером, или только когда кандидат впервые кликнул по ссылке?
8.  **Fallback при падении LLM:** Если API Anthropic/OpenAI недоступно во время завершения сессии, как генерировать Session Summary? Просто сохраняем транскрипт в БД и добавляем кнопку "Сгенерировать AI-отчет позже"?
9.  **Защита от фрода на переводах:** Автоматический перевод контента стоит нам API-кредитов. Нужен ли Hard Limit (дневной лимит) на количество переводов для аккаунта, чтобы избежать злонамеренной генерации 10,000 проектов до списания средств в PayPro?
10. **Конфликты при импорте (Data Overwrite):** Если HR повторно импортирует CSV, где email совпадает, но должность кандидата изменилась — мы молча перезаписываем данные профиля или требуем ручного подтверждения (Ignore/Override)?
