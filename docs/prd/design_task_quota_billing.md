# О проекте (Context & Goals)

Мы внедряем систему лимитов (Quota) на одновременное количество активных слушателей с назначениями (Listeners Seats) и функционал их покупки. Дизайнеру необходимо спроектировать интерфейсы для контроля этих лимитов на стороне пользователя (предупреждение при превышении) и процесс покупки дополнительных мест в разделе биллинга.

**Важно:** Дизайн панели супер-администратора (Super Admin) делать **не нужно**. Задача касается только интерфейса пользователя (презентатора) в основном приложении.

## Где это находится:
1. **Модальные окна:** Всплывающее окно (Overage Modal), которое блокирует процесс создания/шеринга презентации, если лимит исчерпан.
2. **Настройки (Settings):** Вкладка Billing — блок покупки дополнительных мест.
3. **Глобальный UI (или страница Enrollments):** Плашка/прогресс-бар текущего использования квоты.

## Ссылки и референсы

# Что нужно нарисовать (Scope of Work)

Необходимо спроектировать интерфейс контроля и биллинга квот, состоящий из следующих ключевых блоков:

## Блок 1: Модальное окно превышения лимита (Overage Modal)

**Задача:** Прервать флоу создания назначения, доходчиво объяснить пользователю причину блокировки и предложить пути решения (докупить места или освободить текущие).

**Поведение:** Появляется поверх экрана при попытке нажать "Save / Create" в модалке Share / Enroll, если `текущие активные слушатели + новые > лимит`.

**Элементы интерфейса:**
* **Иконка/Иллюстрация:** Предупреждающая, но не агрессивная (например, щит, замок, AlertCircle желтого/оранжевого цвета или кастомная иллюстрация Pitch Avatar).
* **Заголовок:** "Listener Seats Quota Exceeded" (или "Not enough Listener Seats").
* **Пояснительный текст:** "You have reached your limit of **[X]** active listeners. To send new assignments, you need to either purchase additional seats or archive/delete older inactive assignments."
* **Кнопки действий (Footer):**
  * **Primary кнопка (Синяя/Акцентная):** "Upgrade Quota" (ведет на страницу биллинга).
  * **Secondary/Ghost кнопка:** "Cancel" или "Manage existing enrollments" (закрывает окно).

## Блок 2: Покупка дополнительных мест (Settings -> Billing)

**Задача:** Органично встроить калькулятор покупки дополнительных мест (Listeners Seats) в существующую вкладку Billing.

**Поведение:** Интерактивный блок, который мгновенно пересчитывает итоговую сумму в зависимости от выбранного количества мест с учетом оптовой скидки.

**Элементы интерфейса:**
* **Карточка/Секция:** "Buy More Enrollments Quota" с подзаголовком "Purchase additional capacity for Listeners with Assignments."
* **Управление количеством (Input/Slider):**
  * Текст: "How many extra seats do you need?"
  * UI-элемент: Поле ввода числа с кнопками минус [-] и плюс [+] по бокам, либо ползунок (slider) с шагом (например, 10, 50, 100...).
* **Блок расчета цены (Price Summary Box):**
  * Светло-серый или акцентный фон для выделения цифр.
  * Строка 1: "Price per seat" -> `$10.00 / mo` (или `$8.00 / mo` при покупке > 100).
  * Строка 2 (Total): "Total billed monthly" -> Крупным акцентным шрифтом (например, синим) `$100.00`.
  * **Бейдж скидки (Discount hint):** Если выбрано $\le$ 100 мест, выводить зеленую подсказку с иконкой: "Add [X] more to unlock volume discount ($8/seat)".
* **Кнопка действия:** "Proceed to Checkout" (Primary, выровнена по правому краю).

## Блок 3: Индикатор использования квоты (Quota Usage Banner)

**Задача:** Дать пользователю визуальное понимание, сколько мест у него свободно прямо сейчас, чтобы блокировка (Overage) не стала сюрпризом.

**Где разместить (на усмотрение дизайнера, можно предложить оба варианта):**
* Вариант А: Инфо-плашка вверху страницы `/enrollments`.
* Вариант Б: Компактный прогресс-бар в боковом меню (Sidebar) в самом низу.

**Элементы интерфейса:**
* **Текст:** "Listeners Seats Usage: 85 / 100".
* **Прогресс-бар:** Полоска, которая меняет цвет. 
  * Синий/Зеленый — в штатном режиме.
  * Оранжевый/Красный (Warning state) — когда заполнено более 90%.
* **Ссылка/Кнопка:** "Upgrade" (мелкая ссылка рядом с полоской, ведущая в биллинг).

# Definition of Done для Дизайна

1. **Overage Modal:** Отрисованы состояния модального окна превышения лимита.
2. **Billing Tab:** Отрисован блок калькулятора покупки дополнительных мест.
3. **Интерактивные состояния:** Показаны Hover-эффекты для кнопок `+` / `-`, фокус поля ввода, и изменение UI при достижении скидки (от 1 до 100 мест vs более 100 мест).
4. **Индикатор квоты:** Предложено местоположение и дизайн бейджа/прогресс-бара "Listeners Seats Usage" (включая Warning-состояние при заполненности $>90\%$).
5. Макеты финализированы в Figma, организованы логически, связаны в легкий прототип (если нужно показать изменение цены) и готовы к передаче разработчикам (Handoff).
