1.1. Description

Разработка и внедрение базовой сущности "Слушатель" (Listener) для перехода от одноразовых анонимных веб-ссылок к персонализированной системе управления контактами. Эпик охватывает создание раздела Listeners с полным CRUD (создание, чтение, обновление, удаление), функционал массового импорта/экспорта списков контактов в форматах CSV/XLS, а также умный импорт резюме (CV) в форматах PDF/Word с автоматическим извлечением данных (парсингом) с помощью LLM.

1.2. Business Value

*   **Ускорение работы:** Исключение необходимости повторного ручного ввода данных слушателей при каждой отправке презентации. Экономия времени за счет массового импорта и автоматического парсинга резюме.
*   **Глубокая аналитика:** Возможность сбора и просмотра статистики просмотров в разрезе конкретного человека, географии, компании и департамента.
*   **Персонализация:** Аватар сможет использовать сохраненные профильные данные для персонализированного обращения к слушателю.
*   **Синхронизация:** Создание фундамента (сущности Listener) для будущих двусторонних интеграций со сторонними системами (CRM, LMS, ATS).
*   **Новая модель монетизации:** Возможность ввести биллинг на основе количества активных слушателей в системе (Listener Seats).

1.3. Product Structure & Flow

*   Новый пункт меню **Listeners** в боковой навигационной панели.
*   Главная страница раздела представляет собой таблицу всех контактов с возможностью сортировки, поиска и встроенным функционалом редактирования и удаления.
*   Модальное окно редактирования профиля Listener включает поля для персональных данных и область для прикрепления файлов контента (например, резюме).
*   Функционал импорта разделен на два потока:
    1.  Импорт таблицы CSV/XLS.
    2.  Загрузка CV в форматах PDF/Word с последующим LLM-разбором.
*   Экспорт всей базы или выбранных слушателей в CSV/XLS.

3. User Stories (US)

**US-01: CRUD Управление списком Listeners (User)**

Как презентатор, я хочу иметь доступ к списку слушателей, создавать новых, редактировать их данные и удалять неактуальных, чтобы не вводить информацию вручную при каждой рассылке.

*Acceptance Criteria:*
*   В левом меню добавлен пункт "Listeners".
*   На странице выводится список слушателей (таблица) с возможностью сортировки.
*   Доступны действия: добавить нового, редактировать существующего, удалить.
*   При редактировании можно загрузить или удалить прикрепленные файлы контента (резюме, документы).

**US-02: Массовый импорт из CSV/Excel (User)**

Как презентатор, я хочу импортировать список контактов из файлов CSV или XLS, чтобы быстро перенести свою базу без ручного ввода каждого слушателя.

*Acceptance Criteria:*
*   В интерфейсе есть кнопка Import с выбором типа "Table (CSV/XLS)".
*   Пользователь может указать соответствие (маппинг) колонок.
*   Система игнорирует первую строку, если она содержит заголовки.
*   Если импортируется контакт с уже существующим в базе email, система показывает предупреждение и спрашивает: заменить данные (Replace), пропустить (Ignore) и возможность применить это решение ко всем конфликтам (Apply to all).

**US-03: Умный импорт резюме (CV Parser) (User)**

Как презентатор (рекрутер), я хочу загружать файлы PDF или Word с резюме кандидатов (поштучно или массово), чтобы система автоматически распознавала их данные и создавала профили слушателей.

*Acceptance Criteria:*
*   В интерфейсе импорта есть опция "CV (PDF/Word)".
*   Можно загрузить один или несколько файлов одновременно.
*   Система использует LLM для извлечения ключевых данных (Email, Имя, Телефон, Должность и т.д.) из текста файла.
*   Создается карточка слушателя с распознанными данными, а сам оригинальный загруженный файл прикрепляется к профилю.

**US-04: Экспорт базы слушателей (User)**

Как презентатор, я хочу экспортировать свою базу слушателей в формат CSV или Excel, чтобы использовать эти данные во внешних системах.

*Acceptance Criteria:*
*   В интерфейсе доступна кнопка Export.
*   Экспорт формирует файл с правильным порядком колонок согласно стандарту.
*   В будущем: возможность экспортировать только отфильтрованных/выбранных чекбоксами слушателей (пока можно реализовать экспорт всех).
