# 🕵️‍♂️ Global Audit (Мульти-линзовый аудит)

Этот навык инициирует глубокую проверку кодовой базы по 9 независимым направлениям (линзам). 

## 🧠 Линзы (Зоны ответственности)

1. **Линза ①: Консистентность слоёв** — Проверяет, не нарушаются ли границы слоев (например, бизнес-логика в UI-компонентах или прямые запросы к БД в обход сервисов).
2. **Линза ②: Безопасность/IDOR** — Ищет RLS-уязвимости в Supabase, незащищенные API-роуты, утечки PII-данных.
3. **Линза ③: Синхронизация состояния** — Проверяет глобальное состояние (Zustand/Context), инвалидацию кэша и гонки состояний при асинхронных запросах.
4. **Линза ④: Ошибки/Пустота/Оффлайн** — Ищет необработанные `catch`, отсутствие Skeleton-экранов, Empty States и поведение при отвале сети.
5. **Линза ⑤: Жизненный цикл данных** — Проверяет мутации, каскадное удаление (ON DELETE), очистку слушателей (useEffect cleanup) и утечки памяти.
6. **Линза ⑥: Навигация/deep-links** — Анализирует роутинг (Next.js App Router), обработку несуществующих страниц, редиректы авторизации.
7. **Линза ⑦: Инварианты/trust-поля** — Проверяет строгую типизацию (Zod/TypeScript), отсутствие `any`, валидацию форм и проверку входящих данных с клиента.
8. **Линза ⑧: Производительность/Индексы** — Анализирует тяжелые рендеры (memo, useCallback), N+1 запросы к БД, отсутствие индексов в SQL.
9. **Линза ⑨: Конкурентность/гонки** — Проверяет транзакционность (race conditions) при параллельных запросах или обновлениях данных (optimistic updates).

## 🚀 Инструкция для ИИ (Как выполнять)

Когда пользователь вызывает `/global-audit`:
1. Изучи структуру файлов (особенно `src/`, `supabase/`, `components/`).
2. "Надень" по очереди каждую из 9 линз и проанализируй основные файлы через её призму.
3. Сформируй единый Markdown-артефакт `GLOBAL_AUDIT_REPORT.md` с результатами.
4. Подсвети критические угрозы красным (Alerts).
